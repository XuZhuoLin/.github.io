    Hadoop考试复习

    XZLin 2024/1/8 Hadoop

    # Hadoop考试复习。

    # 一、

    (1)大数据的特点4V:
    数据量大，数据速率大，数据多样性大，数据价值密度低。

    (2)谷歌大数据的三驾马车
    GFS：分布式文件系统
    MapReduce：分布式计算框架
    BigTable：分布式数据库

    (3)Hadoop生态圈
    HDFS:是整个生态圈的基石，分布式文件系统，大数据文件存储的基础设施。
    Yarn：分布式计算资源调度框架，为分布式计算MapReduce，spark提供分布式资源调度服务
    MapReduce：分布式计算编程框架，为编写和运行MapReduce程序提供API
    Habse：分布式列族数据库
    Hive：基于Hadoop的数据仓库工具，讲结构化数据文件映射成二维表进行管理
    Zookeeper：分布式数据管理和协调框架，保证分布式环境中的数据的一致性。

    # 二、 Hadoop安装部署

    单机版、分布式（伪分布式，完全分布式）
    1、安装JDK，配置JAVA_HOME,path的环境。
    2、安装oepnssh-server，并且配置免密登录。
    3、安装Hadoop，配置Hadoop_home,path环境
    4、配置Hadoop的环境变量，hadoop-env.sh,core-site.xml,hdfs-site.xml,mapred-site.xml,yarn-site.xml等
    5、初始化Namenode，HDFS namenode -format

    start-all.sh
    start-dfs.sh
    start-yarn.sh
    stop-all.sh
    stop-dfs.sh
    stop-yarn.sh

    # 三、.HDFS

    # （1）NameNode的WebUI以及端口：namenode:50070,端口为9000

    文件系统状态保存：fsimage文件系统镜像备份和edits文件系统编辑日志
    高可用（HA）：主备切换，两台namenode，当主节点挂掉时，切换到备用节点继续工作，主节点和备用节点管理相同的文件系统，同时只有主节点提供服务
    联邦（Federation）：每个namenode独立提供不同的文件系统服务，公用Datanode集群，联邦中的某一个namenode挂掉，不影响其他的namenode

    # （2）secendaryNameNode的webui：secendaryNamenode：50090

    功能：管理检查点check point，把edits中的最新信息同步到fsimage文件中，加速缩短namenode 的启动时间
    通常namenode和secendarynamenode不会部署到同一个服务器
    fsimage和edits文件的合并过程：
    1、生成新的edits.new文件，用于记录合并过程中产生的记录
    2、当触发某一时机时，（时间间隔达到1小时或者edits记录的事务达到一百万条）会从namenode下载edits和fsimage到secendarynamenode，
    3、将两者合并，成为fsimage.ckpt
    4、将fsimage.ckpt上传到namenode
    5、替换掉原来的fsimage
    6、替换掉原来的edits文件

    # （3）DataNode功能：

    1、存储数据块，默认块大小128M
    2、负责客户端对数据库的io请求
    分块存储的好处：
    1、支持大规模文件存储，不受单点存储的上线限制
    2、利于备份
    3、提高吞吐量
    4、负载均衡
    一个数据块会在多个Datanode进行备份，HDFS默认存储3份

    # （4）HDFS shell：

    创建目录：hdfs dfs -mkdir -p
    创建空文件：hdfs dfs -touchz
    查看文件列表：hdfs dfs -ls
    查看文件内容：hdfs dfs -cat
    复制文件：hdfs dfs -cp
    移动文件：hdfs dfs -mv
    删除文件：hdfs dfs -rm
    删除目录：hdfs dfs -rm -r
    上传文件：hdfs dfs -put
    下载文件：hdfs dfs -get
    合并下载：hdfs dfs -getmerge

    # 四、YARN和MapReduce

    yarn是什么：是一个通用的，分布式计算资源调度框架
    yarn的组成部门：
    ResourceManager：资源管理器负责管理集群资源的分配申请
    NodeManager：节点管理器，负责一个计算节点的容器管理
    AppliactionMaster：运行mrappmaster主应用，负责一个mapreduce jop的管理，maptask和reducetask向其报告执行状态
    Application：执行具体的task
    MapReduch是什么：是一个分布式编程框架，具有易于编程，良好的扩展性，高容错性的特点，适合大规模数据的离线处理
    MapReduce的大致工作流程：
    1、对输入的文件进行切片划分
    2、启动相应数量的maptask进程
    3、调用inputFormat中的reduceReader，处理源数据，将数据封装为<k1,v1>键值对
    4、将封装好的<k1,v1>键值对传给mapper
    5、mapper调用自定义的map函数对数据进行计算
    6、计算完成后，将结果封装为<k2,v2>，并输出
    7、框架收集mapper的输出，进行分区和排序shuffle
    8、reduce启动，并从mapper端拉取数据，reduce获取到输入数据<k3,v3>
    9、reduce task调用自定义的reduce函数进行计算，得到输出<k4,v4>
    10、调用outputformat的Recordwriter将结果输出

    # 五、Hbase是什么：

    是一个开源的，分布式的，版本化的非关系型数据库

    # 六、Hive

    数据类型：
    整型：tinyint，smallint，int，bigint
    浮点型：float，double
    字符串：string
    布尔型：bool
    键值对：map
    数组：array
    结构体：struct
    Hive数据结构模型有哪些：数据表、视图、分区、分桶。
    HiveQL：
    建表：create table 001 （name string，age int，number string）
    partitioned by （分区列名 列数据类型） 分区表
    clustered by （分桶列名） into （分桶数量） buckets 分桶表
    row format delimited fields terminated by ‘，’ 指定每行文件格式
    location hdfs_path 指定表文件存储路径
    插入数据表：insert into table 。。。 values。。。
    查询：insert into table1 select from table2
    文件导入数据表：load data inpath 。。。 into table 。。。

    # 七、Zookeeper是什么：

    是一个高可用的分布式数据管理和协调框架，能够很好的保证分布式环境中数据的一致性
    Zookeeper的数据模型：
    zookeeper server端存在一个leader节点，其他节点都是follower
    对外的服务端口默认为2181
    leader主要负责进行投票的发起和决议，更新系统状态
    follower负责接受客户端请求并向客户端返回结果，在选举过程中参与投票
    zookeeper维护着一个树形层次结构，树中节点称为Znode
    每个znode上都会保存自己的数据内容，同时会保存一系列属性信息，且数据不超过1MB。
